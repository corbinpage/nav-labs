Appian 7.8 Review

Corbin Page

Jan. 12, 2015

I’ve been using Appian’s BPM suite for several years now and have seen the product evolve quite a bit over time. Appian generally releases a new version every quarter, and the short release cycle has become even more important since they went “all in” on the Tempo design in Appian 7.0. Version 7.8 was released on December 22nd, 2014 and has a number of features I’m excited about.

Configurable User Start Pages

I’ve heard this feature requested quite often ever since the “5 Tabs” Tempo interface was introduced in 7.2. Now users logging into Appian can be automatically redirected to the specific Task List, Record(s), or Report they use most. So your Support Team will immediately see their open tickets, and your executive sponsor will immediately see her high-level dashboard without needing to navigate to it. The start pages are controlled through the Administration Console and are based off Group membership, providing a simple mechanism for managing different subsets of users.

While I know users will absolutely love this new feature, I can’t help but wonder if configurable start pages will degrade the value of the Tempo Feed. The Feed already faces strong headwinds from 1) older users not comfortable with a “Twitter-like” feed and 2) process designers struggling to strike the right balance between over-posting and under-posting.

In my experience, I have seen few companies wholeheartedly adopt and routinely utilize the Feed. It’s silverlining, however, was that users *had* to view the Feed on initial login, which allowed users to scan recent activity in a more passive manner than email notifications. My guess is that most companies will switch out the Feed as the default start page and opt to display more data-centric views like a Record or Report.

Maturing Following/Followers Model

Whether configurable start pages ends up demoting the Feed remains to be seen, but Appian did add a couple of new features to continue maturing the followers/followees model.

‘Follow Records’ functionality was added as both a Smart Service and as a button that now appears on every single Record Dashboard. Following a record means that you’ll see any post where that specific record is tagged (unless specifically prohibited). I think the Smart Service will help designers keep relevant stakeholders (Account Owners, etc.) updated in an automated manner, but I think the ‘Follow’ button will confuse users into thinking they need to follow every single record. I wish this feature was configurable at the Record-level.

Additionally, the ‘Add Followers’ Smart Service has been renamed to ‘Follow Users Smart Service’ for consistency and is more forgiving when attempting to follow a secure user.

Updated Interface Designer

The 7.8 release also delivered the largest improvement to the SAIL Interface Designer since its introduction in 7.5. The ‘point and click’ UI design is back!

Appian’s legacy forms designer had a pretty robust ‘point and click’, WYSIWYG editor, but SAIL forms, so far, have always required coding to create. I personally really enjoy writing SAIL but understand that 1) business analysts (and less technical process designers) are intimidated by writing “code” and 2) developers don’t love learning a proprietary front-end language before they can start building their first UIs.

Enter the Design View.

The Design View was introduced in 7.8 to bring back the ‘point and click’ model for creating forms, records, and reports. Click to insert a section, a couple text fields, some checkboxes, etc. and you’ll see the live view update in real time. Even better, while you’re modifying a component in the Design View, the interface will highlight that section to help you keep track of your modifications. And for power users, you can easily toggle back and forth between the Expression View that contains the editable SAIL code (in all its glory).

I’ve always compared the Interface Designer to a “lite” version of Developer Tools in Chrome and Firebug for Firefox. Now those tools are more robust of course (meant for all-out front-end development), but the Interface Designer needs to fulfill the same role for Appian’s streamlined interface. This first version of the Design View looks robust enough to handle most simple-to-medium complexity interfaces, but I’d guess it’s easier to flip to the Expression View for using advanced expressions like applyComponent() and servlet plugins.

That said, I think the Design View absolutely gets Business Analysts back to being comfortable building Appian UIs and making tweaks. And that’s a big step.

Best of the Rest

  • Query Process Analytics
    • Would be a bigger deal if the ‘Portal to SAIL’ plugin did not exist. Nice to see Appian supporting this functionality OOTB though.
  • Paragraph fields in Grid Layouts
    • Grids are frequently used to edit lists of child CDTs on forms, and regular text fields never provide enough information for editing longer fields like descriptions or comments. So instead, the common design pattern was to select a row of the grid and edit it in a dynamically appearing section below the grid.
    • Supporting the paragraph field in Grid Layouts introduces a new, more “Excel-like” option for building out this feature.
  • Mobile
    • All mobile links (including task links in email notifications) open up in the iOS or Android app, instead of the website in your default browser.
    • Mobile offline connectivity notification was added to iOS. Android coming soon.
  • Process Model constants
    • Replaces Process Model UUID as the best way to reference a process model in an expression.
  • Site Access Administration Page
    • More awesome functionality added to the Administration Console.
    • If you use Appian for authentication, you can now specify minimum password strength, account locking, and several other features.
  • Concurrent Sessions
    • The maximum number of concurrent sessions per account can now be set (but turned off by default).

So..Should I Upgrade?

If you’re running Appian on the cloud, you’ll be prompted to upgrade no matter what. So this section isn’t for you. If you’re running Appian on-premise, Configurable User Start Pages might be important enough to justify an upgrade to your Business Sponsor. Or perhaps the new ‘point and click’ Interface Designer would significantly help your new process designers get ramped up on the product. However, for clients unwilling to upgrade every single quarter, I’d suggest waiting until the next release (usually in April) since the pre-Appian World release is always feature-packed.

Hope you enjoyed my little rundown of Appian 7.8! Send all questions, comments, and critiques to @corbpage! I’d love to discuss!

Until 7.9!
