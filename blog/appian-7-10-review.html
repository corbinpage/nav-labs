Appian 7.10 Review

Corbin Page

Jul. 13, 2015

Between the summer-y days of Fourth of July weekend and this past weekend, you might have missed that Appian 7.10 was released! Although 7.10 is a relatively minor release compared to the pre-Appian World 7.9 release, it still has plenty of improvements to work into your custom enterprise apps.

Themes for Embedded Interfaces

Embedded Interfaces now support custom branding to seamlessly integrate into your customer-facing website or enterprise portal. I love the continued focus on Embedded Interfaces, which allow one product (Appian) to manage the entire process of an external request, notification, triage, and resolution. For example, you could have a ‘Contact Us’ or ‘Request Demo’ form on your website that posts to the Appian feed once submittes and assigns a task to your BizDev team to reach out to the prospective customer. All without a single integration! No need to string different systems or APIs together!

I’ve actually built custom solutions for this exact same workflow for several different startup clients. It usually involves connecting a forms-builder like Typeform or Wufoo with a middle-tier service that pushes data into Salesforce, Workday, or an ERP system. Now you can do all that work with one Appian process model! Pretty cool!

Objects View

The Objects View allows all objects in an environment to be viewed in the Appian Designer regardless of whether they are part of an application. This View is great to have OOTB, and I’ve seen clients maintain an “All Objects” application to accomplish the same purpose.

Video SAIL Component

Similar to how Appian already allows external image content to be served dynamically from a Content Delivery Network (CDN) or other source, video content is now also available. Various video file types are supported on different devices with MP4 being the most widely supported. I’m not entirely sure how I’d use this component on a regular basis, but I love that it’d make Records and Report even more dynamic displays. Tweet me any novel uses of video you end up building!

Unused Process Model Version Cleanup

If you’re in the Appian Process Modeller as much as I am, you have the Ctrl+Alt+S shortcut engrained in your muscle memory. When developing a process model, you should Save & Publish very frequently. But these process model versions build up quickly over time and used to cause the execution engines to get large on a Dev or QA environment. So much so that there were plug-ins developed to delete these old versions. But now Appian clears the old versions out of memory itself. Woohoo!

Best of the Rest

  • Default Values for Interfaces and Expression Rules – great addition to the Interface Designer and expression rules
  • New design to Milestone component – The milestone component is now blue, instead of green. Your end-users might notice but likely won’t mind.
  • Editable grid – now supports Date Times, radio buttons, and progress bars
  • More SAIL performance improvements (always a good thing )

Should I Upgrade?

Probably not, unless you’re on a cloud install or are dying to add a theme to your Embedded Interfaces. If you upgrade twice a year, then I’d recommend waiting for the next release.

A full list of updates can be found in the release notes. Send all questions and comments to @corbpage! I’d love to discuss what you think of the release!

I’ll be back for my slurpee in Appian “7.11”. See you then!
